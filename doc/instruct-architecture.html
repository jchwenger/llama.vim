llama.vim — Instruct-Based Editing

Natural-language code refactoring powered by a local LLM

1 · Overview

Select code in visual mode, press <leader>lli, type an instruction, and the LLM returns the refactored block — streamed in real time. You can accept, cancel, rerun, or continue the conversation.

Core idea — The selected text, its surrounding context, and accumulated ring-buffer chunks are assembled into a structured chat prompt. The model is instructed to return only the transformed code block, which replaces the original selection.

3 · Prompt Construction

llama#inst_build() assembles a system prompt with four context regions and a user message containing the instruction.

System Message
You are a text-editing assistant. Respond ONLY with the result of applying
INSTRUCTION to SELECTION given the CONTEXT. Maintain the existing text
indentation. Do not add extra code blocks.

↓ structured sections follow ↓
── CONTEXT ──────────────────────────────────────────
Ring buffer chunks (recently viewed / yanked / edited code)
function foo() { ... }
class Bar { ... }
── PREFIX ───────────────────────────────────────────
Lines before the selection (up to n_prefix = 256)
import { something } from 'module';
const x = 42;
── SELECTION ────────────────────────────────────────
The highlighted code block (l0 … l1)
← this is what gets replaced →
── SUFFIX ───────────────────────────────────────────
Lines after the selection (up to n_suffix = 64)
return result;
}
User Message
INSTRUCTION: extract the validation logic into a separate function
For continuation, inst_prev holds the full conversation (system + all prior user/assistant turns), so the model retains context across multiple refinement steps.

7 · Request Object Schema

Each request is stored in s:inst_reqs[id]:

let l:req = {
    \ 'id':         req_id,       " unique incrementing ID
    \ 'bufnr':      bufnr('%'),   " buffer number
    \ 'range':       [l0, l1],     " [start_line, end_line]
    \ 'status':      'proc',       " proc → gen → ready
    \ 'result':      '',           " accumulated generated text
    \ 'inst':        'instruction', " current instruction text
    \ 'inst_prev':   [],           " full conversation history
    \ 'job':         v:null,       " curl job handle
    \ 'n_gen':       0,            " token count
    \ 'extmark':     -1,           " selection highlight extmark
    \ 'extmark_virt': -1,         " virtual text extmark
    \ }

8 · Key Implementation Details

  • Warm-up request — llama#inst() sends a zero-predict request with the context assembled before the user types the instruction. The server caches the prompt, so the actual generation starts faster.
  • Streaming SSE — s:inst_on_response() parses data: {...} lines, extracts choices[0].delta.content, and appends to req.result in real time.
  • Low temperature — The instruct request uses min_p=0.1, temperature=0.1 for deterministic, precise refactoring output.
  • Conversation history — inst_prev is a standard chat messages array. On continuation, the new instruction is appended and the full history is resent, enabling iterative refinement.
  • Position tracking — llama#inst_update_pos() queries the extmark position to keep req.range accurate when edits happen nearby.
  • Dual rendering — Neovim uses nvim_buf_set_extmark() with virt_lines; Vim 9.0+ uses prop_add() with text_align: 'after'.
  • Accept callback — s:inst_callback() strips trailing empty lines, deletes the original range via deletebufline(), and inserts the result with append().
